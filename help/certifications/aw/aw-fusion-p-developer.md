# Certifieringsresa för Adobe [!DNL Workfront Fusion] Developer Professional

Lär dig hur du blir certifierad Adobe Developer Professional.

## Minimal upplevelse

Du borde ha:

* Förmåga att [!DNL Workfront] API- och API-utforskaren, men behöver inte nödvändigtvis vara effektiv vid API-anrop
* Möjligheten att använda anpassade API-anrop för att arbeta med [!DNL Workfront] och andra API:er
* Möjligheten att använda moduler för [!DNL Workfront] program och andra åtgärder som att skapa, läsa, uppdatera, ta bort och söka i
* Möjlighet att bygga livskraftiga lösningar som ger affärsresultat utan att ge avkall på [!DNL Fusion] skyddsräcken
* Möjlighet att välja lämplig utlösarmodul för att uppnå affärsresultat
* En förståelse för hur flödeskontroll används (vägar, iteratorer/aggregatorer, förklara flöde)
* Möjlighet att utföra grundläggande felsökning och felhantering
* Möjlighet att effektivt testa och leva med ett scenario
* Möjlighet att hantera organisationsteam och användare
* En förståelse för syftet med och användningen av universella anslutningar för REST API Oauth2

>[!NOTE]
>
>Din certifiering gäller i två år från och med när du godkänns. Vi rekommenderar att du använder den nyare versionen när det är möjligt eftersom den omfattar nyare produktfunktioner.

**Målgrupp:**

* Utvecklare, automatiseringstekniker
* Automatisering, processoptimering och automatisering

* Systemadministratör (IT-, MIS-, teknik-, Knowledge Systems-, AI-administratörer)

**Information om tentamen:**

* Nivå: Professionell (0-12 månaders erfarenhet)
* Lösenpoäng: 33/51
* Tid: 102 min
* Leverans: Bearbetad online (kräver kameraåtkomst) eller testcenter
* Tillgängliga språk: Engelska
* Kostnad: $125 (global) / $95 (Indien)
* Exam-ID: AD0-E902

**Mål och omfattning för tentamen**

Avsnitt 1: Grundläggande tekniska begrepp (39 %)

* Med [!DNL Fusion] scenario med ett fältformat som måste ha ett annat fältformat, tillhandahålla möjliga funktioner som kan användas för att omforma data korrekt
* Med [!DNL Fusion] scenario där en funktion måste kapslas i en annan funktion, väljer du rätt formaterat uttryck
* Identifiera sätt att använda team för att styra åtkomsten till anslutningar och andra [!DNL Fusion] funktioner
* Med [!DNL Fusion] scenario, identifiera system- och funktionsbegränsningar
* Med [!DNL Fusion] scenario som kräver tidszonsändring, skillnad mellan användare, synlig organisation och var det behövs att åsidosätta tidszonen
* Med [!DNL Fusion] scenario, välj lämplig [!DNL Workfront] modul(er) eller åtgärd(er)
* Med [!DNL Fusion] scenario väljer du det uttryck och/eller de moduler på mappningspanelen som ska omforma indata på rätt sätt till utdata
* Identifiera rätt sätt att använda [!DNL Fusion] Utvecklingsverktyg för felsökning av exekveringsfel eller för att fastställa anrop och svar till tredjepartssystem
* Med [!DNL Fusion] scenario där data i en sökväg behövs i en annan sökväg, tillhandahålla en set-get-lösning för att göra dessa data tillgängliga på ytterligare routningssökvägar
* Identifiera olika sätt att använda eller producera en [!DNL Fusion] mall för ett affärsbehov
* Med [!DNL Fusion] scenario, identifiera vad som ska göras för att visa körningar och/eller åtgärda fel
* Identifiera de olika alternativen i [!DNL Fusion] scenarioinställningar för att hantera scenariobehov
* Med [!DNL Fusion] välj lämplig flödeskontroll och/eller fastställa rätt kombination av flödeskontrollmodulerna

Avsnitt 2: Scenariodesign och arkitektur (35 %)

* Välj rätt sekvens för att tolka JSON och konvertera till paket i en lista med steg
* Ange rätt metod för datasökning utifrån en uppsättning krav
* Identifiera skillnader mellan direktutlösare och schemalagda utlösare
* Identifiera en korrekt modul för att överföra ett dokument till [!DNL Workfront]
* Ange det korrekta tillvägagångssättet för att spåra ett affärsbehov [!DNL Fusion] har bearbetat en post (inte en användare) och när
* Med [!DNL Fusion] scenario, identifiera en möjlighet att förenkla utformningen och/eller optimera för underhåll
* Identifiera [!DNL Fusion] bästa sättet att dokumentera scenarier
* Identifiera en möjlighet att minska dataflödet genom ett scenario
* Förklara ändringarna i antalet åtgärder som bearbetas och visas av paketkontrollen med hjälp av ett exempel på antalet paket som visas i en sökmodul och en efterföljande aggregator
* Identifiera information som finns i paketkontrollen
* Beskriver ett behov av att arkivera ett scenario längre än 60 dagar och väljer det enda möjliga sättet (import/export-plan)
* Ett exempel på ett fel 403: Ej tillåtet, markera felmeddelandets ursprung och orsak

Avsnitt 3: Testning och felhantering (16 %)

* Identifiera elementen i en testplan
* Identifiera elementen i ett testfall
* Med [!DNL Fusion] scenario, identifiera rätt direktiv för att hantera en otillförlitlig tjänst
* Identifiera processen för att lägga till anpassad felhantering i en [!DNL Fusion] scenario
* Identifiera hur du spårar, läser och löser ofullständiga körningar
* Med [!DNL Fusion] scenario där nödvändiga data saknas, välj sätt att hantera ogiltiga data

Avsnitt 4: Arbeta med API:er (10 %)

* Anges ett tredjeparts-API som returnerar 429: För många förfrågningar, identifiera en lösning för att förhindra eller hantera felet
* Med [!DNL Fusion] scenario där nya funktioner inte är tillgängliga i [!DNL Workfront] identifierar du rätt referens och modultyp för den nya funktionen
* Om det finns ett tredjepartssystem som inte har någon dedikerad app identifierar du HTTP-appen och väljer lämplig modul
* Identifiera skillnaden mellan CRUD-åtgärder och andra gemensamma funktioner i REST API:er
